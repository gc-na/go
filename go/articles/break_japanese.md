# Go言語における「break」ステートメントの解説

## 概要
Go言語における「break」ステートメントは、ループやswitch文の実行を中断するために使用されます。このコマンドを利用することで、特定の条件に基づいてループから早期に脱出することが可能です。

## ドキュメンテーション
### 目的
「break」ステートメントは、forループ、switch文、またはselect文の実行を終了させるために使用されます。これにより、条件が満たされた場合にループを中断し、次の処理へ進むことができます。

### 使用法
「break」は以下のように使用されます。

```go
for i := 0; i < 10; i++ {
    if i == 5 {
        break // iが5のときにループを中断
    }
    fmt.Println(i)
}
```

このコードは、0から4までの数を出力し、iが5に達した時点でループを終了します。

### 詳細
- **スコープ**: 「break」は、直近のループまたはswitch文に対してのみ効果があります。
- **ラベル付きbreak**: Goでは、ラベルを使用して特定のループを指定して中断することもできます。ラベルは「`labelName:`」の形式で付けられ、以下のように使用します。

```go
outerLoop:
for i := 0; i < 3; i++ {
    for j := 0; j < 3; j++ {
        if i == 1 && j == 1 {
            break outerLoop // outerLoopを中断
        }
        fmt.Println(i, j)
    }
}
```

この場合、`i`が1で`j`が1の時に、外側のループを中断します。

## 例
以下は「break」の基本的な使用例です。

```go
package main

import "fmt"

func main() {
    for i := 0; i < 10; i++ {
        if i%2 == 0 {
            fmt.Println("偶数:", i)
        } else {
            break // 奇数が見つかったらループを中断
        }
    }
}
```

このプログラムは、最初の偶数を出力した後、最初の奇数が見つかった時点でループを終了します。

## 説明
- **一般的な落とし穴**: 「break」を使用する際は、意図したループやスコープを正確に把握しておくことが重要です。特にラベル付きの「break」を使用する場合、誤って異なるループを中断するリスクがあります。
- **ネストされたループ**: 複数のネストされたループがある場合、どのループを中断するかを明確にするためにラベルを使用することが推奨されます。

## 一言まとめ
Go言語の「break」ステートメントは、ループやswitch文の実行を特定の条件で中断するための便利な機能です。
