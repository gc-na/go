# نوع البيانات "int" في لغة Go: دليل شامل

## ملخص
يعتبر نوع البيانات "int" في لغة Go من الأنواع الأساسية التي تستخدم لتخزين الأعداد الصحيحة. يعد "int" مثاليًا للاستخدام في العمليات الحسابية والتكرارات، وهو عنصر أساسي في تطوير البرمجيات بلغة Go.

## التوثيق
### الغرض
يتم استخدام نوع البيانات "int" في لغة Go لتخزين الأعداد الصحيحة، حيث يمثل مجموعة من القيم التي يمكن استخدامها في العمليات الحسابية والتكرارية. يختلف حجم "int" حسب بنية النظام، حيث يكون عادةً 32 بت على الأنظمة 32 بت و64 بت على الأنظمة 64 بت.

### الاستخدام
يمكن استخدام نوع "int" في التصريحات، العمليات الحسابية، والمقارنات. يتم التصريح عن متغير من نوع "int" باستخدام الكلمة المفتاحية `var` أو باستخدام التعريف المختصر `:=`.

### التفاصيل
- **النطاق**: يعتمد نطاق نوع "int" على بنية النظام، حيث يكون:
  - 32 بت: من -2,147,483,648 إلى 2,147,483,647
  - 64 بت: من -9,223,372,036,854,775,808 إلى 9,223,372,036,854,775,807
- **التخزين**: يتم تخزين القيم في الذاكرة بشكل فعّال، مما يجعله مناسبًا للعمليات الحسابية السريعة.
- **العمليات**: يمكن إجراء جميع العمليات الحسابية الأساسية مثل الجمع، الطرح، الضرب، والقسمة.

## الأمثلة
### مثال 1: التصريح عن متغير من نوع "int"
```go
package main

import "fmt"

func main() {
    var x int = 10
    fmt.Println(x)
}
```

### مثال 2: العمليات الحسابية
```go
package main

import "fmt"

func main() {
    a := 5
    b := 3
    sum := a + b
    fmt.Println("Sum:", sum)
}
```

### مثال 3: استخدام "int" في الحلقات
```go
package main

import "fmt"

func main() {
    for i := 0; i < 5; i++ {
        fmt.Println("Iteration:", i)
    }
}
```

## الشرح
### الأخطاء الشائعة
- **تجاوز النطاق**: إذا حاولت تخزين قيمة تتجاوز نطاق "int"، ستحدث أخطاء أو نتائج غير متوقعة.
- **استخدام "int" مع الأنواع الأخرى**: إذا تم استخدام "int" مع أنواع بيانات أخرى (مثل `float64`)، يجب الانتباه إلى تحويل الأنواع لتجنب الأخطاء.
- **عدم التحقق من القيم**: في بعض الأحيان، قد تؤدي العمليات الحسابية إلى قيم غير متوقعة، لذا من المهم التحقق من النتائج.

## ملخص في جملة واحدة
نوع البيانات "int" في لغة Go هو نوع أساسي لتخزين الأعداد الصحيحة، ويستخدم بشكل واسع في العمليات الحسابية والتكرارية.
