# Go语言中的常量（const）详解

## 概述
在Go语言中，`const`关键字用于定义常量。常量是不可修改的值，具有固定的类型和一旦定义后就不再改变的特性。使用常量可以提高代码的可读性和可维护性，同时避免魔法数字的出现。

## 文档
### 目的
常量用于定义那些在程序运行期间不会改变的值，例如数学常数、配置选项或其他不需要动态计算的值。它使得代码更加清晰，避免了硬编码的值。

### 用法
在Go中，常量的定义使用`const`关键字，常量可以是数字、字符串或布尔值。常量可以在包级别或函数级别定义，且可以使用类型推导。

#### 语法
```go
const 常量名 = 值
```

### 细节
1. **类型**：常量可以显示指定类型，也可以让Go进行类型推导。
2. **多重定义**：可以在一行中定义多个常量。
3. **iota**：在常量组中，可以使用`iota`生成一系列连续的常量值。

### 示例
```go
package main

import "fmt"

// 定义常量
const Pi = 3.14
const Greeting = "Hello, World!"

// 使用iota定义常量组
const (
    A = iota // 0
    B = iota // 1
    C = iota // 2
)

func main() {
    fmt.Println("常量 Pi:", Pi)
    fmt.Println("常量 Greeting:", Greeting)
    fmt.Println("常量 A:", A)
    fmt.Println("常量 B:", B)
    fmt.Println("常量 C:", C)
}
```

## 说明
在使用常量时，有几个常见的陷阱和注意事项：
- **类型不匹配**：常量在不同的上下文中可能会产生类型不匹配的错误。例如，将一个整型常量用于需要浮点型的场合。
- **iota的使用**：`iota`在常量组中会自动递增，确保使用时明白其递增逻辑，避免意外的常量值。
- **无效的常量表达式**：常量的值必须是编译时可确定的表达式，动态计算的值不能用于常量定义。

## 一行总结
Go语言中的`const`关键字用于定义不可修改的常量，提高代码的可读性和可维护性。
