# Go 프로그래밍 언어에서의 goto 문: 사용법과 주의사항

## 개요
Go 언어에서 `goto` 문은 프로그램 흐름을 특정 레이블로 점프하는 데 사용됩니다. 이 기능은 코드의 특정 부분으로 직접 이동할 수 있는 방법을 제공하지만, 코드 가독성을 떨어뜨릴 수 있으므로 신중하게 사용해야 합니다.

## 문서화

### 목적
`goto` 문은 조건에 따라 프로그램의 흐름을 제어할 수 있는 방법을 제공합니다. 일반적으로 반복문이나 조건문에서 복잡한 흐름 제어가 필요할 때 사용됩니다.

### 사용법
`goto` 문은 다음과 같은 형식으로 사용됩니다:

```go
goto label
```

여기서 `label`은 코드의 특정 위치에 정의된 레이블입니다. 레이블은 다음과 같이 정의할 수 있습니다:

```go
label:
    // 코드 블록
```

### 세부사항
- `goto` 문은 동일한 함수 내에서만 사용할 수 있습니다.
- 레이블은 문자, 숫자, 언더스코어로 시작해야 하며, 한 줄의 코드 앞에 정의되어야 합니다.
- `goto`를 남용하면 코드의 가독성이 떨어질 수 있으며, 일반적으로는 다른 흐름 제어 구조(예: `if`, `for`, `switch`)를 사용하는 것이 더 권장됩니다.

## 예제

### 기본 사용 예제
```go
package main

import "fmt"

func main() {
    i := 0

    start:
        if i < 5 {
            fmt.Println(i)
            i++
            goto start
        }
}
```
위의 예제에서는 `goto` 문을 사용하여 `start` 레이블로 점프하며, `i`가 5보다 작을 때까지 값을 출력합니다.

### 조건부 점프 예제
```go
package main

import "fmt"

func main() {
    for i := 0; i < 10; i++ {
        if i == 5 {
            goto end
        }
        fmt.Println(i)
    }

end:
    fmt.Println("종료")
}
```
이 예제는 `i`가 5가 되었을 때 `end` 레이블로 점프하여 "종료"를 출력합니다.

## 설명
`goto` 문은 프로그램의 흐름을 제어하는 데 유용할 수 있지만, 다음과 같은 함정이 있습니다:
- **가독성 저하**: 코드가 복잡해지고 이해하기 어려워질 수 있습니다.
- **디버깅 어려움**: 프로그램의 흐름을 추적하기 어려워져 디버깅이 힘들어질 수 있습니다.
- **대체 구조 사용 권장**: `goto` 대신 `if`, `for`, `switch` 같은 다른 제어 구조를 사용하는 것이 일반적으로 더 바람직합니다.

## 한 줄 요약
Go 언어의 `goto` 문은 코드 흐름을 레이블로 점프할 수 있게 해주지만, 가독성을 떨어뜨릴 수 있으므로 신중하게 사용해야 합니다.
