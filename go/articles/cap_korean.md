# Go의 cap: 용량(capacity) 이해하기

## 개요
Go 언어에서 `cap`은 슬라이스, 맵, 채널의 용량을 반환하는 내장 함수로, 데이터 구조의 메모리 할당량을 이해하는 데 중요한 역할을 합니다.

## 문서화
`cap` 함수는 슬라이스, 맵, 채널과 같은 데이터 구조의 최대 용량을 반환합니다. 이 용량은 해당 데이터 구조가 현재 할당 가능한 최대 요소 수를 나타내며, 메모리 관리 및 성능 최적화에 유용합니다.

### 용도
- 슬라이스, 맵, 채널의 최대 용량을 확인할 수 있습니다.
- 동적 메모리 할당의 효율성을 높이기 위해 용량을 사전에 파악할 수 있습니다.

### 사용 방법
`cap` 함수는 아래와 같은 형식으로 사용됩니다:

```go
cap(variable)
```
여기서 `variable`은 슬라이스, 맵, 또는 채널이어야 합니다.

## 예제
### 슬라이스의 용량 확인
```go
package main

import (
	"fmt"
)

func main() {
	slice := make([]int, 0, 5) // 길이는 0, 용량은 5
	fmt.Println(cap(slice)) // 출력: 5
}
```

### 채널의 용량 확인
```go
package main

import (
	"fmt"
)

func main() {
	channel := make(chan int, 3) // 용량이 3인 채널
	fmt.Println(cap(channel)) // 출력: 3
}
```

### 맵의 용량 확인
```go
package main

import (
	"fmt"
)

func main() {
	m := make(map[string]int)
	fmt.Println(cap(m)) // 출력: 컴파일 에러, 맵은 cap 함수 사용 불가
}
```
> 주의: 맵의 경우 `cap` 함수를 사용할 수 없습니다. 이점에 유의해야 합니다.

## 설명
`cap` 함수는 슬라이스와 채널에서는 유용하게 사용되지만, 맵에서는 사용할 수 없습니다. 슬라이스의 경우, 초기 용량을 설정하지 않으면 기본값은 0이며, 후에 요소를 추가할 때마다 자동으로 증가합니다. 채널의 경우, 버퍼링을 통해 용량을 조절할 수 있습니다. `cap` 함수를 사용하여 이러한 용량을 사전에 이해하고 적절한 메모리 관리를 할 수 있습니다.

### 일반적인 실수 및 주의사항
- 맵에 대해서 `cap`을 호출하면 컴파일 오류가 발생합니다.
- 슬라이스의 길이(`len`)와 용량(`cap`)은 서로 다르며, 용량이 길이를 초과할 수 있습니다.
- 용량을 초과하여 요소를 추가하면, 새로운 메모리가 할당되고 기존 요소가 복사됩니다. 따라서 성능 저하를 유의해야 합니다.

## 한 줄 요약
Go 언어의 `cap` 함수는 슬라이스와 채널의 최대 용량을 반환하며, 메모리 관리를 최적화하는 데 중요한 역할을 합니다.
