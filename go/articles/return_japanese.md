# Go言語の「return」文：効果的な関数の終了方法

## 概要
Go言語における「return」文は、関数から値を返し、関数の実行を終了するための重要な構文です。この文を使用することで、関数の結果を呼び出し元に渡すことができます。

## ドキュメンテーション
### 目的
「return」文は、関数の実行を終了し、必要に応じて一つ以上の値を返すために使用されます。これにより、関数の処理結果を利用することが可能になります。

### 使い方
「return」文は、関数の中で使用され、次のように記述します。

```go
func 関数名(引数型 引数名) 戻り値の型 {
    // 処理内容
    return 戻り値
}
```

具体的には、戻り値を指定しない場合は、関数が終了するだけで、値は返されません。戻り値を指定した場合は、指定された型の値を返します。

### 詳細
- **戻り値の型**: 関数の定義時に戻り値の型を指定することができます。複数の戻り値を定義することも可能です。
- **無名戻り値**: 戻り値の型を指定しただけの無名戻り値もサポートされており、関数内で戻り値を直接変更することができます。
- **defer文との併用**: 「defer」文を使用して、関数の終了時に特定の処理を実行することができます。この場合、「return」文が実行された後でも、deferで指定した処理が実行されます。

## 例
以下は「return」文の基本的な使用例です。

### 単一の戻り値を持つ関数

```go
package main

import "fmt"

func add(a int, b int) int {
    return a + b
}

func main() {
    result := add(5, 3)
    fmt.Println(result) // 出力: 8
}
```

### 複数の戻り値を持つ関数

```go
package main

import "fmt"

func divide(a, b int) (int, int) {
    return a / b, a % b
}

func main() {
    quotient, remainder := divide(10, 3)
    fmt.Println(quotient, remainder) // 出力: 3 1
}
```

## 説明
- **戻り値の型の不一致**: 戻り値の型が関数の定義と一致しない場合、コンパイルエラーが発生します。
- **多重return文**: 同じ関数内で複数の「return」文を使用することができますが、最初の「return」文が実行されると、以降の文は無視されます。
- **deferとの注意点**: 「defer」文は、関数が終了する際、最後に実行されるため、戻り値が変更される可能性があることに注意が必要です。

## 一文の要約
Go言語の「return」文は、関数から値を返し、関数の実行を終了するために使用される重要な構文です。
