# Go 语言中的 "package" 详解

## 摘要
在 Go 语言中，"package" 是代码组织和重用的基本单位。它允许程序员将相关的功能和数据组合在一起，以实现模块化和结构化编程。

## 文档
### 目的
Go 语言中的包（package）是将代码组织成可重用模块的机制。每个包可以包含多个源文件，这些源文件可以定义变量、常量、类型和函数。通过使用包，开发者可以更方便地管理大型项目，提高代码的可读性和可维护性。

### 用法
在 Go 中，使用 `package` 关键字定义包。以下是基本的包定义格式：

```go
package packageName
```

- `packageName` 是包的名称，通常与文件夹名称相匹配。
- 每个 Go 文件都必须以 `package` 声明开始。

### 细节
1. **导入包**：要使用其他包中的功能，需要使用 `import` 关键字。例如：
    ```go
    import "fmt"
    ```
2. **标准库**：Go 语言提供了丰富的标准库，涵盖了各种功能，如输入输出、字符串处理、网络等。
3. **自定义包**：用户可以创建自定义包，方法是在一个新文件夹中创建 Go 文件，并在文件中定义包名。
4. **包的可见性**：以大写字母开头的标识符是导出的（可以被其他包引用），而以小写字母开头的标识符则是私有的（只能在本包内使用）。

## 示例
以下是一个简单的 Go 包示例：

```go
// mathutil.go
package mathutil

// Add 返回两个整数的和
func Add(a int, b int) int {
    return a + b
}

// main.go
package main

import (
    "fmt"
    "path/to/your/mathutil" // 替换为实际包路径
)

func main() {
    sum := mathutil.Add(3, 5)
    fmt.Println("Sum:", sum)
}
```

## 说明
- **常见陷阱**：确保包名称与文件夹名称一致，以避免导入时的混淆。
- **循环依赖**：避免在包之间形成循环依赖，这可能导致编译错误。
- **命名冲突**：在导入多个具有相同标识符的包时，使用别名避免命名冲突。

## 一句话总结
在 Go 语言中，"package" 是将相关功能组织在一起的基本单位，促进代码的模块化和重用。
