# الكلمة الرئيسية "return" في لغة Go: الاستخدامات والأمثلة

## ملخص
تُستخدم الكلمة الرئيسية "return" في لغة Go لإنهاء دالة وإرجاع قيمة أو قيم منها. إنها أداة أساسية في التحكم في تدفق البرنامج وتحديد النتائج التي تُرجعها الدوال.

## الوثائق
### الغرض
تُستخدم الكلمة "return" في Go لإنهاء تنفيذ الدالة الحالية وإرجاع قيمة. يمكن استخدام "return" في كل من الدوال التي تُرجع قيمة واحدة أو عدة قيم.

### الاستخدام
تظهر الكلمة "return" في دالة Go على النحو التالي:
```go
func اسم_الدالة() نوع_القيمة {
    // تنفيذ بعض التعليمات
    return قيمة
}
```
إذا كانت الدالة تُرجع قيمتين، يمكن كتابتها كالتالي:
```go
func اسم_الدالة() (نوع_القيمة1, نوع_القيمة2) {
    // تنفيذ بعض التعليمات
    return قيمة1, قيمة2
}
```

### التفاصيل
- يمكن استخدام "return" في أي مكان داخل الدالة.
- عند الوصول إلى "return"، يتم إنهاء الدالة، ويتم إرجاع القيم المحددة.
- إذا كانت الدالة لا تُرجع أي قيمة، يمكن استخدام "return" بدون أي قيمة.

## الأمثلة
### مثال 1: دالة بسيطة تُرجع قيمة واحدة
```go
package main

import "fmt"

func جمع(a int, b int) int {
    return a + b
}

func main() {
    نتيجة := جمع(3, 4)
    fmt.Println(نتيجة) // الناتج: 7
}
```

### مثال 2: دالة تُرجع قيمتين
```go
package main

import "fmt"

func تقسيم(a int, b int) (int, int) {
    return a / b, a % b
}

func main() {
    الناتج, الباقي := تقسيم(10, 3)
    fmt.Println("الناتج:", الناتج, "الباقي:", الباقي) // الناتج: 3 الباقي: 1
}
```

## الشرح
### الأخطاء الشائعة
- **عدم إرجاع القيمة المطلوبة**: يجب التأكد من أن جميع مسارات تنفيذ الدالة تُرجع قيمة إذا كان ذلك مطلوبًا. إذا لم يتم ذلك، سيظهر خطأ في الترجمة.
- **استخدام "return" في دالة بلا قيمة**: في الدوال التي لا تُرجع قيمة، يمكن استخدام "return" بدون أي قيمة، لكن يمكن أن يسبب ذلك ارتباكًا إذا لم يكن واضحًا.

### ملاحظات إضافية
- يمكن استخدام "return" في الدوال المسماة أيضًا، حيث يتم استخدام أسمائها بدلاً من النوع.
- يمكن أن تحتوي الدوال على "return" متعددة، ولكن يجب أن تتطابق القيم المعادة مع أنواع القيم المعلنة.

## ملخص جملة واحدة
تُستخدم الكلمة الرئيسية "return" في لغة Go لإنهاء الدوال وإرجاع قيمة أو قيم منها، مما يسهل التحكم في تدفق البرنامج.
