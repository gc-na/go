# uint16 في لغة Go: نوع بيانات صحيح غير موقع

## ملخص
`uint16` هو نوع بيانات صحيح غير موقع (Unsigned Integer) في لغة Go، ويستخدم لتخزين الأعداد الصحيحة التي تتراوح قيمتها بين 0 و65535.

## الوثائق
### الغرض
يُستخدم `uint16` لتخزين الأعداد الصحيحة غير السالبة التي تتطلب مساحة تخزين صغيرة، مما يجعله خياراً مثالياً في التطبيقات التي تحتاج إلى كفاءة في استخدام الذاكرة.

### الاستخدام
يمكن تعريف متغير من نوع `uint16` بسهولة كالتالي:
```go
var number uint16
```
يمكن أيضاً تعيين قيمة مباشرة:
```go
number := uint16(500)
```
### التفاصيل
- **النطاق**: النطاق القيم الممكنة لـ `uint16` هو من 0 إلى 65535.
- **الذاكرة**: يستخدم `uint16` 2 بايت من الذاكرة.
- **العمليات**: يمكن إجراء العمليات الحسابية الأساسية مثل الجمع، الطرح، الضرب، والقسمة.

## الأمثلة
### مثال 1: تعريف واستخدام `uint16`
```go
package main

import "fmt"

func main() {
    var x uint16 = 1000
    var y uint16 = 2000
    sum := x + y
    fmt.Println("مجموع:", sum) // الناتج: مجموع: 3000
}
```

### مثال 2: التعامل مع حدود `uint16`
```go
package main

import "fmt"

func main() {
    var max uint16 = 65535
    fmt.Println("أقصى قيمة لـ uint16:", max) // الناتج: أقصى قيمة لـ uint16: 65535

    // محاولة إضافة 1 إلى الحد الأقصى
    overflow := max + 1
    fmt.Println("Overflow:", overflow) // الناتج: Overflow: 0
}
```

## الشرح
### الفخاخ الشائعة
- **تجاوز السعة**: عند محاولة إضافة قيمة أكبر من الحد الأقصى لـ `uint16`، سيحدث تجاوز (Overflow) ويعود إلى القيمة 0.
- **عدم دعم القيم السالبة**: لا يمكن تخزين القيم السالبة، لذا يجب التأكد من أن جميع القيم التي يتم تعيينها لـ `uint16` غير سالبة.

### ملاحظات إضافية
- يعتبر `uint16` مفيدًا في التطبيقات التي تحتاج إلى تخزين أعداد صحيحة صغيرة، مثل معالجة البيانات الرقمية أو التشفير.
- على الرغم من كونه فعالاً في استخدام الذاكرة، إلا أنه يجب توخي الحذر عند التعامل مع العمليات الرياضية التي قد تؤدي إلى تجاوز السعة.

## ملخص جملة واحدة
`uint16` هو نوع بيانات صحيح غير موقع في Go، يستخدم لتخزين الأعداد الصحيحة من 0 إلى 65535 بكفاءة في الذاكرة.
