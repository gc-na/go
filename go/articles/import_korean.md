# Go 언어의 "import" 사용법: 패키지 가져오기

## 개요
Go 언어에서 `import`는 다른 패키지를 현재 파일에 포함시키는 데 사용되는 키워드입니다. 이를 통해 코드의 재사용성을 높이고, 기능을 모듈화하여 관리할 수 있습니다.

## 문서화
`import` 문은 Go 프로그램에서 외부 패키지를 가져오는 데 필수적입니다. Go는 표준 라이브러리와 사용자 정의 패키지를 포함하여 다양한 패키지를 제공합니다. `import`를 사용함으로써 개발자는 필요한 기능을 손쉽게 사용하고, 코드의 가독성을 높일 수 있습니다.

### 사용법
`import` 문은 파일 상단에 위치해야 하며, 여러 패키지를 가져올 때는 괄호를 사용하여 그룹화할 수 있습니다. 기본적인 형식은 다음과 같습니다:

```go
import "패키지_경로"
```

또는 여러 패키지를 가져올 때는 다음과 같이 사용할 수 있습니다:

```go
import (
    "패키지_경로1"
    "패키지_경로2"
)
```

### 세부 사항
- 각 패키지는 고유한 경로를 가지고 있으며, Go의 설정된 GOPATH에 따라 관리됩니다.
- 패키지 이름은 경로의 마지막 부분으로 결정되며, 이는 코드에서 참조할 때 사용됩니다.
- Go는 `import` 문에 대한 중복을 허용하지 않으며, 이미 가져온 패키지는 다시 가져올 필요가 없습니다.

## 예제
1. 기본 패키지 가져오기:
   ```go
   package main

   import "fmt"

   func main() {
       fmt.Println("Hello, World!")
   }
   ```

2. 여러 패키지 가져오기:
   ```go
   package main

   import (
       "fmt"
       "math"
   )

   func main() {
       fmt.Println(math.Sqrt(16))
   }
   ```

3. 사용자 정의 패키지 가져오기:
   ```go
   package main

   import (
       "fmt"
       "yourmodule/mypackage"
   )

   func main() {
       mypackage.MyFunction()
   }
   ```

## 설명
- **패키지 경로**: 패키지 경로는 Go 모듈에 따라 다를 수 있으며, 올바른 경로를 사용할 수 있도록 주의해야 합니다.
- **가져오기 실패**: 경로가 잘못되거나 패키지가 존재하지 않으면 컴파일 오류가 발생합니다.
- **패키지 이름 충돌**: 동일한 이름의 패키지를 여러 번 가져오면 충돌이 발생할 수 있습니다. 이 경우 별칭을 사용하여 문제를 해결할 수 있습니다.

## 한 줄 요약
Go 언어에서 `import` 키워드는 외부 패키지를 가져와 코드의 재사용성을 높이는 데 필수적인 기능이다.
