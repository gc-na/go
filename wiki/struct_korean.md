# Go 언어의 구조체 (Struct): 개념과 활용

## 개요
Go 언어에서 구조체(Struct)는 데이터의 집합을 정의하는 사용자 정의 타입으로, 관련된 데이터를 그룹화하여 더 복잡한 데이터 구조를 만들 수 있게 합니다. 구조체는 Go의 기본 데이터 유형을 조합하여 새로운 데이터 유형을 생성하는 데 사용됩니다.

## 문서화
구조체는 Go 프로그래밍에서 중요한 역할을 하며, 객체 지향 프로그래밍의 특성을 일부 지원합니다. 구조체는 필드(field)로 구성되며, 각 필드는 특정 타입의 데이터를 가집니다. 구조체를 사용하면 관련 데이터를 하나의 단위로 묶어 관리하고 조작할 수 있습니다.

### 구조체의 정의
구조체는 `type` 키워드를 사용하여 정의합니다. 다음은 기본적인 구조체 정의 문법입니다:

```go
type StructName struct {
    FieldName1 FieldType1
    FieldName2 FieldType2
    // ...
}
```

### 구조체의 사용
구조체를 선언한 후, 이를 사용하여 새로운 인스턴스를 생성할 수 있습니다. 인스턴스 생성은 리터럴 문법을 사용하거나 `new` 키워드를 사용하여 할 수 있습니다.

```go
person := StructName{FieldName1: value1, FieldName2: value2}
```

### 구조체의 포인터
구조체의 포인터를 사용하면 메모리 효율성을 높이고, 구조체의 필드를 수정할 때 복사본을 만들지 않도록 할 수 있습니다.

```go
personPointer := &person
```

## 예제
다음은 구조체를 정의하고 사용하는 간단한 예제입니다:

```go
package main

import "fmt"

// 사람을 나타내는 구조체 정의
type Person struct {
    Name string
    Age  int
}

func main() {
    // 구조체 인스턴스 생성
    p1 := Person{Name: "Alice", Age: 30}
    
    // 구조체 필드 접근
    fmt.Println("이름:", p1.Name)
    fmt.Println("나이:", p1.Age)

    // 구조체 필드 수정
    p1.Age = 31
    fmt.Println("수정된 나이:", p1.Age)
}
```

## 설명
구조체를 사용할 때 주의해야 할 몇 가지 사항이 있습니다:

1. **필드의 가시성**: 구조체 필드는 대문자로 시작하면 공개(public)되고, 소문자로 시작하면 비공개(private)됩니다. 이는 패키지 간 접근 제어에 영향을 미칩니다.
   
2. **구조체 복사**: 구조체는 전송 시 기본적으로 복사됩니다. 큰 구조체를 함수나 메서드에 전달할 때는 포인터를 사용하는 것이 성능에 더 유리합니다.
   
3. **임베디드 구조체**: Go에서는 다른 구조체를 포함하는 형태로 구조체를 임베드할 수 있습니다. 이를 통해 코드의 재사용성을 높일 수 있습니다.

## 한 줄 요약
Go 언어에서 구조체(Struct)는 관련 데이터를 그룹화하여 사용자 정의 데이터 유형을 생성할 수 있는 강력한 도구입니다.
