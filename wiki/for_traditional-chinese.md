# Go 語言中的「for」迴圈：用法與範例

## 摘要
在 Go 語言中，「for」是唯一的迴圈控制結構，能夠用來實現各種迴圈需求，包括傳統的計數迴圈、條件迴圈和迭代範圍。這個關鍵字簡潔而強大，適用於多種場景。

## 文件說明
「for」迴圈是 Go 語言中用來重複執行程式碼的基本結構。它具有三種主要的形式：

1. **基本迴圈**：沒有條件，會持續執行，直到被顯式中止。
2. **計數迴圈**：使用初始化語句、條件語句及增量語句來控制迴圈的執行。
3. **範圍迴圈**：遍歷陣列、切片、映射或通道等資料結構。

### 語法
```go
for 初始化; 條件; 增量 {
    // 執行的程式碼
}
```
或
```go
for 條件 {
    // 執行的程式碼
}
```
或
```go
for 變數, 值 := range 資料結構 {
    // 執行的程式碼
}
```

## 範例
### 基本迴圈
```go
package main

import "fmt"

func main() {
    i := 0
    for {
        if i >= 5 {
            break
        }
        fmt.Println(i)
        i++
    }
}
```

### 計數迴圈
```go
package main

import "fmt"

func main() {
    for i := 0; i < 5; i++ {
        fmt.Println(i)
    }
}
```

### 範圍迴圈
```go
package main

import "fmt"

func main() {
    fruits := []string{"蘋果", "香蕉", "橘子"}
    for index, fruit := range fruits {
        fmt.Printf("%d: %s\n", index, fruit)
    }
}
```

## 解釋
使用「for」迴圈時，有一些常見的陷阱和注意事項：

- **無限迴圈**：若沒有適當的終止條件，可能會導致無限迴圈，這在調試時可能會很麻煩。
- **範圍迴圈的變量重用**：在範圍迴圈中，變量會在每次迭代時重新賦值，這可能會引起一些意外行為，尤其是在閉包中使用時。
- **變量作用域**：在「for」迴圈中宣告的變量僅在該迴圈內有效，這對於控制變量的作用域是有益的。

## 一句話總結
Go 語言的「for」迴圈是實現重複執行程式碼的唯一結構，具有靈活的用法和多樣的迴圈形式。
