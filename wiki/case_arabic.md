# استخدام عبارة "case" في لغة Go: دليلك الشامل

## ملخص
تعتبر عبارة "case" في لغة Go جزءًا أساسيًا من التحكم في التدفق، حيث تستخدم في عبارات "switch" لتحديد حالات متعددة وتنفيذ كود معين بناءً على القيم المدخلة.

## الوثائق
عبارة "case" تُستخدم ضمن عبارة "switch" لتحديد الشروط المختلفة التي يمكن من خلالها اتخاذ قرارات برمجية. تُعتبر "switch" بديلاً أكثر وضوحًا ومرونة لعدة عبارات "if" المتداخلة.

### الغرض
تساعدك عبارة "case" على تنظيم الشروط بشكل أفضل وتسهيل قراءة الكود، مما يؤدي إلى تقليل التعقيد وتحسين الصيانة.

### الاستخدام
تستخدم عبارة "case" داخل عبارة "switch" كما يلي:

```go
switch variable {
case value1:
    // تنفيذ الكود إذا كانت variable تساوي value1
case value2:
    // تنفيذ الكود إذا كانت variable تساوي value2
default:
    // تنفيذ الكود إذا لم تتطابق مع أي من القيم السابقة
}
```

## أمثلة
### مثال 1: استخدام عبارة "case" مع قيم عددية

```go
package main

import "fmt"

func main() {
    number := 2

    switch number {
    case 1:
        fmt.Println("العدد هو 1")
    case 2:
        fmt.Println("العدد هو 2")
    case 3:
        fmt.Println("العدد هو 3")
    default:
        fmt.Println("العدد غير معروف")
    }
}
```

### مثال 2: استخدام عبارة "case" مع قيم نصية

```go
package main

import "fmt"

func main() {
    fruit := "تفاح"

    switch fruit {
    case "موز":
        fmt.Println("الاختيار هو موز")
    case "تفاح":
        fmt.Println("الاختيار هو تفاح")
    case "برتقال":
        fmt.Println("الاختيار هو برتقال")
    default:
        fmt.Println("فاكهة غير معروفة")
    }
}
```

## الشرح
### الأخطاء الشائعة
- **عدم وجود حالة `default`**: إذا لم يتم تحديد حالة `default`، فلن يتم تنفيذ أي كود إذا لم تتطابق القيمة مع أي من الحالات المحددة.
- **عدم استخدام الأقواس**: يجب استخدام الأقواس `{}` بعد عبارات "case" لتنظيم الكود بشكل صحيح.
  
### ملاحظات إضافية
- يمكن استخدام تعبيرات متعددة في حالة واحدة عن طريق فصلها بفواصل.
- يمكن استخدام "fallthrough" لتجاوز الحالة التالية، لكن يجب استخدامه بحذر.

## ملخص جملة واحدة
عبارة "case" في لغة Go تُستخدم داخل عبارة "switch" لتحديد الشروط المختلفة لتنفيذ كود معين بناءً على القيم المدخلة.
