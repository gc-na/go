# Go 語言中的 int8 資料型別詳解

## 概述
`int8` 是 Go 語言中的一種整數資料型別，專為需要占用較小記憶體空間的整數數據而設計。它的值範圍為 -128 到 127，適合用於儲存小範圍的整數數據。

## 文檔

### 目的
`int8` 資料型別的主要目的是提供一種能夠用於儲存整數的高效方式，特別是在記憶體有限的情況下。它的使用可以幫助開發者優化程式的記憶體使用。

### 使用方式
在 Go 中，`int8` 可以通過以下方式聲明與使用：

```go
var a int8 = 100
```

上述代碼宣告了一個 `int8` 型別的變數 `a`，並將其初始化為 100。您也可以通過計算或函數返回值來賦值。

### 詳細說明
- **範圍**：`int8` 的範圍是 -128 到 127，因此如果試圖賦值超出這一範圍，將會導致編譯錯誤。
- **記憶體佔用**：`int8` 佔用 1 個字節（8 位元），這使得它在需要大量整數變數的情況下非常優化。
- **運算**：`int8` 支持基本的算術運算，如加法、減法、乘法和除法，但需注意運算後的結果仍需符合 `int8` 的範圍。

## 範例

以下是一些基本的 `int8` 使用範例：

```go
package main

import "fmt"

func main() {
    var a int8 = 100
    var b int8 = 20
    var sum int8 = a + b // 正確，結果在範圍內
    fmt.Println("Sum:", sum)

    var c int8 = 127
    // var d int8 = c + 1 // 這行會導致編譯錯誤，因為超出範圍
    fmt.Println("Value of c:", c)
}
```

## 解釋

### 常見陷阱
1. **範圍超出**：在進行算術運算時，應特別注意結果是否超出 `int8` 的範圍，否則會導致運算錯誤或意外行為。
2. **類型轉換**：在與其他整數型別（如 `int` 或 `int16`）進行運算時，建議明確進行類型轉換，以避免類型不匹配的錯誤。

### 其他注意事項
- `int8` 不支持浮點運算，若需要進行浮點數計算，應使用 `float32` 或 `float64`。
- 在進行數據序列化或網路傳輸時，使用 `int8` 能夠有效減少傳輸的數據大小。

## 一句總結
`int8` 是 Go 語言中一種高效的整數型別，適合用於儲存小範圍的整數數據，佔用1個字節的記憶體。
