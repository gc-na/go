# Go 語言中的 uint16 整數型別詳解

## 簡介
在 Go 語言中，`uint16` 是一種無符號整數型別，佔用 2 個位元組（16 位元），其值範圍從 0 到 65535。這使得 `uint16` 特別適合需要節省內存或僅需表示小範圍數值的場景。

## 文檔
### 目的
`uint16` 是 Go 語言內建的基本數據類型之一，主要用於表示無符號整數，適合需要正整數且範圍有限的應用。

### 使用方法
在 Go 中，`uint16` 可以直接使用，定義變量的方法如下：

```go
var x uint16
```

你也可以在宣告的同時給變量賦值：

```go
var x uint16 = 500
```

或者使用類型推斷：

```go
x := uint16(500)
```

### 詳細說明
- **記憶體佔用**：`uint16` 每個變量佔用 2 個位元組，適合在需要高效內存使用的情況下使用。
- **範圍**：`uint16` 的值範圍是 0 到 65535，所以當你需要表示的數值超過此範圍時，應考慮使用更大的整數型別，例如 `uint32` 或 `uint64`。
- **數學運算**：`uint16` 之間可以進行數學運算，但要注意結果可能會超出 `uint16` 的範圍，這時將會發生溢出，導致結果不正確。
- **類型轉換**：在使用 `uint16` 時，需小心與其他數字類型的轉換，特別是從 `int` 或其他整數類型轉換時，必須考慮到範圍問題。

## 範例
以下是 `uint16` 的基本用法示例：

```go
package main

import "fmt"

func main() {
    var a uint16 = 30000
    var b uint16 = 10000
    var sum uint16 = a + b // 正確，sum 為 40000

    fmt.Println("Sum:", sum)

    // 示範溢出
    var c uint16 = 60000
    var d uint16 = 10000
    var overflow uint16 = c + d // 將發生溢出
    fmt.Println("Overflowed Sum:", overflow) // 會顯示 65535 以上的錯誤結果
}
```

## 解釋
- **常見陷阱**：使用 `uint16` 時，開發者需要特別注意數值的範圍，避免溢出。例如，在做加法運算時，如果結果超過 65535，將會發生溢出，導致錯誤的計算結果。
- **類型轉換**：在進行類型轉換時，應確認來源類型的值在 `uint16` 的範圍內，否則會造成數據丟失或錯誤。

## 一句總結
`uint16` 是 Go 語言中的一種無符號整數型別，適合用於存儲小範圍的正整數，具有高效的內存使用特性。
